# 日新教师 AI 工作台

从真实教学任务出发，帮助教师找到对应工具、提示词与使用边界。

这个项目不是单纯的链接导航页，而是面向小学教师、一线教师培训和校本教研的 AI 工作入口。首页尽量保持清爽，让老师快速找到工具或提示词。

## 当前包含

- 网页工具：适合直接打开使用的 GitHub Pages 或其他网页入口。
- 妙搭应用：适合课堂互动、写作反馈、课例展示等具体教学场景。
- Skill / 能力包：更适合维护者或工作流间接使用的资料处理能力。
- 聚合入口：例如“日新教师工具箱”，用于进入多个子模块。
- 提示词工作台：用 `prompts.json` 管理常用提示词，支持查看详情和复制。

## 本地预览

本项目是静态网页，不依赖构建工具。进入项目目录后运行：

```bash
python -m http.server 4173 --bind 127.0.0.1
```

然后打开：

```text
http://127.0.0.1:4173
```

也可以直接打开 `index.html`，但建议使用本地服务预览，避免浏览器对本地 JSON 读取的限制。

## 数据文件

### tools.json

工具数据配置文件。新增工具前先确认这些信息：

- `type`：工具类型，例如 `webpage`、`miaoda`、`skill`、`hub`、`pending`。
- `url`：正式访问链接。不能确认可访问时不要硬填。
- `status`：状态，例如 `online`、`pending`、`internal`、`draft`。
- `platform`：运行平台，例如 GitHub Pages、妙搭、Skill / 能力包。
- `privacyLevel`：隐私提醒级别。
- `usageBoundary`：使用边界，说明适合谁用、怎么用、不能替代什么。
- `recommendedPrompts`：关联提示词 id。

新增工具时不要直接堆卡片。先确认类型、链接、状态、使用边界，再决定是否放到首页。

### prompts.json

提示词数据配置文件。提示词正文不写死在 HTML 中，便于后续维护。

每条提示词应包含标题、分类、适用平台、使用场景、输入材料、完整提示词、输出格式、使用提醒、标签和关联工具。

### pending-inputs.md

待人工补充和确认的清单，包括：

- 待补充链接
- 待补充提示词
- 待确认工具
- 后续发布清单

不确定的信息统一放入这里，不要写成正式内容。

## 维护原则

- 首页只放精简信息，完整说明放在详情弹窗。
- 未确认链接不要伪装成正式入口。
- 外部工具、妙搭应用和 Skill / 能力包要区分清楚。
- 涉及学生信息、作文、评语、家庭沟通内容时，必须提醒教师脱敏和人工审核。
- 新增功能前先判断是否破坏当前信息层级，避免首页变成卡片堆。

## 发布前检查

建议每次发布前运行：

```bash
node -e "JSON.parse(require('fs').readFileSync('tools.json','utf8')); JSON.parse(require('fs').readFileSync('prompts.json','utf8')); console.log('JSON ok')"
node --check app.js
git diff --check
```
